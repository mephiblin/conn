# 레거시 스킬 시스템

이 문서는 과거 웹게임의 스킬/주사위 loadout 시스템을 Unity 재개발 참고용으로
정리한다. JavaScript 코드를 그대로 포팅하지 않고, 같은 규칙을 Unity C#의 스킬,
스킬덱, 전투 주사위 모델로 다시 구현한다.

## 참조 소스 위치

## 핵심 개념

스킬 시스템은 주사위 loadout 기반이다. 캐릭터는 여러 개의 6면 주사위를 가지며,
각 면에는 스킬 카드와 눈 값이 들어간다.

전투 턴이 시작되면 각 주사위에서 하나의 면이 `roll`로 생성된다. 플레이어는
슬롯 릴처럼 회전하는 결과를 STOP으로 확정한 뒤 제한 개수만큼 선택한다. 선택된
roll은 `value`, `skillId`, `kind`, `targetMode`, `formula`, `effect`, `cooldown`을
기준으로 공격, 회복, 방어, 버프, 디버프, 흡혈 등의 효과로 처리된다.

장착되지 않은 면은 fallback 기본공격으로 해석된다. 쿨다운은 스킬 ID가 아니라
`dieId::faceIndex` 단위로 적용되어, 같은 스킬이라도 어느 주사위 면에 나왔는지에
따라 독립적으로 관리된다.

## 기본 규칙

- 기본 주사위 수는 5개다.
- 기본 선택 한도는 3개다.
- 각 주사위는 6개의 면을 가진다.
- 각 면은 `value`와 `skillId`를 가진다.
- 전투 턴 시작 시 각 주사위에서 하나의 면이 뽑혀 `roll`이 된다.
- 플레이어는 STOP으로 릴을 멈춘 뒤, 멈춘 결과 중 선택 한도만큼 고른다.
- 선택한 roll들은 순서대로 공격, 회복, 방어, 버프, 디버프, 흡혈 등으로 해석된다.
- 비어 있는 면은 `기본공격`으로 대체된다.
- `signature` 면은 캐릭터의 직업/대표 스킬로 치환된다.
- 쿨다운은 스킬 전체가 아니라 `dieId::faceIndex` 슬롯별로 관리된다.

## 스킬 데이터

스킬 정의는 최소한 다음 필드를 가진다.

- `id`: 스킬 식별자
- `name`: 표시 이름
- `kind`: 효과 종류
- `targetMode`: 대상 방식
- `effect`: 공식에 들어가는 효과값
- `formula`: 주사위 값과 효과값을 조합하는 공식
- `cooldown`: 사용 후 슬롯 쿨다운
- `duration`: 버프/디버프 지속 턴
- `status`: 부여할 상태 이상
- `tags`: 분류 태그
- `buyPrice`: 구매 가격
- `sellPrice`: 판매 가격
- `catalogIds`: 스킬 상점 재고 분류
- `description`: 설명

Unity에서는 이 데이터를 `SkillDefinition` ScriptableObject 또는 빌드용 DTO로
관리한다. 제작은 Unity Editor Tool에서 하고, 런타임은 검증된 스킬 정의만 읽는다.

## 스킬덱과 주사위 면

스킬덱은 일반 카드 덱이 아니라 캐릭터의 주사위 면 배치다.

필요한 런타임 데이터는 다음과 같다.

- 캐릭터가 보유한 스킬 목록
- 캐릭터가 장착 중인 주사위 목록
- 각 주사위의 6개 면
- 각 면의 눈 값
- 각 면에 장착된 스킬 ID
- 각 면 슬롯의 쿨다운 상태

스킬 장착 UI는 보유 스킬을 특정 주사위의 특정 면에 배치하는 화면이어야 한다.
전투 중에는 이 loadout을 읽어 릴 결과를 생성한다.

## 기술 상인

마을의 기술 상인은 스킬 시스템과 직접 연결된다.

기술 상인은 다음 기능을 제공한다.

- 판매 가능한 스킬 목록 표시
- 스킬 구매
- 스킬 판매
- 플레이어 보유 스킬 표시
- 스킬 설명, 공식, 쿨다운, 대상 방식 표시
- 구매한 스킬을 스킬덱에 장착할 수 있는 흐름 연결

상점 재고는 고정 목록 또는 catalog 기반 랜덤 목록이 될 수 있다. 레거시 웹게임은
`catalogIds`와 NPC 서비스 정의를 통해 기술 상인 재고를 구성했다. Unity에서도
초기에는 catalog 기반 재고를 유지하는 편이 데이터 확장에 유리하다.

## 효과 종류

레거시 스킬은 다음 효과 종류를 사용한다.

- `attack`: 적에게 피해
- `guard` / `defend`: 방어와 가드 수치 부여
- `heal`: 아군 회복
- `support`: 적 약점 노출
- `buff`: 아군 강화
- `debuff`: 적 약화
- `lifesteal`: 피해 후 일부 회복
- `summon`: 소환 계열, 현재는 보류 로직

## 대상 방식

대상 방식은 스킬 효과가 누구에게 적용되는지를 결정한다.

- `enemy`: 적 1명
- `ally`: 아군 1명
- `self`: 자기 자신
- `party`: 파티 전체 또는 파티 중 조건 대상
- `all_enemies`: 모든 적

## 공식

스킬 공식은 주사위 값과 스킬 효과값을 조합한다.

- `die_as_effect`: 주사위 값 그대로
- `die_plus_effect`: 주사위 값 + 효과값
- `die_minus_effect`: 주사위 값 - 효과값
- `die_times_effect`: 주사위 값 x 효과값
- `die_divide_effect`: 주사위 값 / 효과값
- `die_equals_effect`: 고정 효과값

Unity에서는 공식 문자열을 그대로 퍼뜨리지 말고 enum과 evaluator로 분리한다.

## Unity 구현 방향

초기 C# 구조는 다음처럼 나눌 수 있다.

- `SkillDefinition`: 스킬 기본 데이터
- `SkillInventory`: 캐릭터가 보유한 스킬 목록
- `DiceLoadout`: 캐릭터가 장착한 주사위 구성
- `DiceDefinition`: 6면 주사위 데이터
- `DiceFaceDefinition`: 주사위 한 면의 눈 값과 스킬 ID
- `SkillSlotCooldown`: `dieId::faceIndex` 단위 쿨다운
- `SkillFormulaEvaluator`: 공식 계산
- `SkillEffectResolver`: 효과 종류별 적용
- `SkillDeckController`: 스킬덱 UI와 장착/해제 입력
- `SkillShopService`: 기술 상인 거래 처리
- `SkillShopController`: 기술 상인 UI

## 첫 구현 범위

초기 vertical slice에서 필요한 스킬 시스템은 다음까지만 구현한다.

1. 직업 대표 스킬 1개를 캐릭터에 부여
2. 기본 주사위 5개 생성
3. 각 주사위 6면에 기본 스킬 또는 빈 면 배치
4. 전투 시작 시 roll 5개 생성
5. STOP 후 최대 3개 선택
6. 선택된 스킬의 공식과 효과 적용
7. 슬롯별 쿨다운 적용
8. 기술 상인은 두 번째 단계에서 구매/판매와 스킬덱 장착 UI로 확장
