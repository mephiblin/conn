# 추가 결정 사항

이 문서는 구현 전 의문으로 남아 있던 항목에 대한 현재 결정을 기록한다.

## 1. 조우 트리거

던전 전투 진입은 보이는 몬스터와 접촉하는 방식으로 시작한다.

- 던전 안에 몬스터가 시각적으로 배치된다.
- 플레이어가 몬스터와 접촉하면 `Dungeon -> Combat`으로 전환한다.
- 전투 종료 후 `Combat -> Dungeon`으로 복귀한다.

초기 구현은 몬스터 오브젝트의 collider/trigger 접촉으로 처리한다.

## 2. 퀘스트 완료 후 탈출

퀘스트 목표를 달성하면 즉시 팝업으로 선택지를 보여준다.

- `귀환`
- `계속 탐험`

`계속 탐험`을 선택하면 HUD에 귀환 버튼을 활성화한다. 이후 플레이어는 원하는 시점에
귀환 버튼으로 마을에 돌아가 정산할 수 있다.

## 3. 사망 처리

초기 구현에서 사망은 즉시 Ending으로 처리한다.

- 파티/주인공 사망
- 퀘스트 실패
- Ending 또는 패배 화면으로 전환

아이템 손실, 골드 손실, 마을 회수 같은 세부 패널티는 이후 확장한다.

## 4. 무기별 주사위 개수

전투에서 굴리는 주사위 개수는 장착 무기가 정한다.

| 장비 상태 | 주사위 수 | 추가 효과 |
| --- | ---: | --- |
| 한손무기 | 4 | 기본 |
| 한손무기 + 방패 | 3 | 방어 보정 |
| 양손무기 | 5 | 공격 중심 |
| 무기 없음 | 2 | 최소 전투 가능 |

방패의 방어 보정 수치는 이후 밸런싱 단계에서 정한다.

## 5. 자동 지도

자동 지도는 지나간 영역의 fog를 해제하는 방식으로 구현한다.

- 자유 이동 FPS를 유지한다.
- 에디터 격자/구역 데이터를 기반으로 지도 영역을 나눈다.
- 플레이어가 지나간 영역만 지도에 표시한다.

셀 스냅 이동은 사용하지 않는다.

## 6. Generated Item 범위

첫 구현은 고정 아이템만 사용한다.

단, generated item 구조는 문서와 데이터 계약에 남겨둔다. 이후 장비 상점과 전리품에
다음 요소를 확장할 수 있다.

- rarity
- affix pool
- generated offer
- 감정
- 저주

## 7. 마을 NPC UI 깊이

초기 마을 NPC는 전부 더미 대화를 제공한다.

단, 다음 기능은 첫 단계에서 실제 구현 대상으로 둔다.

- 기술 상인: 스킬 구매/판매
- 대장장이: 장비 구매/판매

게시판과 게이트는 vertical slice 필수 기능이다.

## 8. 훈련소 역할

훈련소 역할은 아직 보류한다.

초기에는 더미 대화만 제공한다. 성장, 능력치 상승, 스킬 숙련도, 주사위 관련 강화는
이후 별도 설계에서 결정한다.

## 9. 학자 역할

학자 역할은 아직 보류한다.

초기에는 더미 대화만 제공한다. 감정, 저주 해제, 퀘스트 힌트, 세계관 정보 제공은
이후 별도 설계에서 결정한다.

## 10. 퀘스트 보상

초기 퀘스트 보상은 골드만 지급한다.

이후 확장 후보는 다음과 같다.

- 고정 아이템
- 랜덤 전리품
- generated item
- 상점 재고 리롤
- 상점 재고 해금
- NPC 서비스 해금
