# 레거시 에디터 참조 목록

이 문서는 Unity Editor Tool을 만들 때 참고할 레거시 웹게임 에디터 파일 목록이다.
레거시 소스 루트는 [`webgame_ref/`](../webgame_ref/)이다.

목표는 웹 에디터를 그대로 이식하는 것이 아니다. Unity에서는 인게임 에디터를
만들지 않고, Unity Editor의 `EditorWindow`, Inspector, Custom Inspector,
ScriptableObject 편집 화면으로 다시 구현한다. 따라서 아래 파일들은 UI 복제가
아니라 기능 책임과 데이터 계약을 뽑아가는 용도로만 사용한다.

에디터는 게임 모드가 아니다. 실제 게임 씬과 분리된 제작 파이프라인이며, 게임에
쓰이는 데이터들을 생성하고 관리한 뒤 검증된 산출물을 Runtime에 넘긴다. 제작용 씬을
사용할 수는 있지만 release player build에는 포함하지 않는다.

## 1. 에디터 전체 구조

- [`webgame_ref/src/renderEditor.js`](../webgame_ref/src/renderEditor.js):
  웹 에디터 UI의 메인 조립 파일. `Generator Workbench`와 `Legacy Cell Editor`가
  어떻게 나뉘는지 참고한다.
- [`webgame_ref/src/editorState.js`](../webgame_ref/src/editorState.js):
  에디터 상태 구조, 선택값, 탭 상태, 편집 중인 데이터의 기준이다.
- [`webgame_ref/src/editorBindings.js`](../webgame_ref/src/editorBindings.js):
  버튼/입력 이벤트가 실제 편집 로직에 어떻게 연결되는지 보는 파일이다.

Unity 대응:

- `EditorWindow` 진입점
- 선택 중인 에셋/맵/노드 상태
- Inspector 입력과 데이터 변경 연결

## 2명 워커 기준 분리

Editor Tool의 첫 목표는 전체 에디터 완성이 아니다. 먼저 Runtime vertical slice에
필요한 데이터를 만들 수 있어야 한다. 따라서 1차 목표는 다음 순서로 둔다.

1. `Content Database`: 몬스터, 아이템, 장비, 스킬, 퀘스트, vendor, 상점 catalog,
   rarity, affix pool 데이터 import/편집
2. `Build & Validation`: ID 중복, 누락 참조, 퀘스트-던전-조우 연결 검증
3. `Map Editor`: 자유 이동 FPS 던전용 방/복도/오브젝트/조우 배치 데이터 작성
4. `Generator`: 맵 초안 생성과 authored map 저장
5. `NPC/Quest`: 마을 NPC 서비스, 게시판 퀘스트, 게이트 연결 편집
6. `Event Graph`: 첫 vertical slice 이후 확장

셀/격자 기반 편집은 플레이어 이동 방식을 의미하지 않는다. Unity Runtime의 던전과
마을은 자유 이동 FPS 공간이며, 에디터의 셀/격자는 맵 생성과 오브젝트 배치 기준이다.

## 최종 EditorWindow 구성

Unity에서는 최종적으로 다음 EditorWindow들을 둔다.

- `Map Editor`
- `Generator`
- `Content Database`
- `Event Graph`
- `NPC/Quest`
- `Build & Validation`

이 창들은 플레이어 Game Runtime에 포함하지 않는다. 모두 `Conn.Editor` assembly에
두고, Runtime은 Editor가 출력한 검증 완료 데이터만 읽는다.
